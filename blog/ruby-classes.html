Got Class?

September-25th-2015

Well do you? punk! A class in ruby is a way of saying, HEY! I want this to be a large object. You can use it to make a code that is re usable all over your code. You could make a People class that has all the things needed to describe a general person. Then you could call on this class and add in details to make it so that each new Person you create will be unique.

What do I mean by that? Well let us break this down. We create a new class called person. In this new class we define methods inside the class that will describe what this person does. So each person will have a name, a job, and a home. We can make a method for all of these different things. We can then create a new person named Bob, give him a job of coder, and a home of the Whitehouse (President coder!) and it would use the built in methods to spit these things out. Now we could create a new person name Nob. Now nob has everything in a person but he has a different name and job and house. He walks like bob, he talks like bob, but he is not bob. So why would we spend all the time trying to re write all that walking/talking code for Nod? Thanks to the person class we do not have to! This is the joy of classes. We can create a base model for things that are similar and use that over and over again and just make small changes to the individual things we want.
